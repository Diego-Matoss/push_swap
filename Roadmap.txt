Paso 1: main.c
Flujo mental:
main
 ├── si argc == 1 → salir sin imprimir nada
 ├── parsear argumentos
 ├── crear stack a
 ├── si ya está ordenado → liberar y salir
 ├── si size == 2 → sort_two
 ├── si size == 3 → sort_three
 └── liberar memoria
Todavía no metas --simple, --medium, --complex, --adaptive. Puedes dejarlo preparado mentalmente, pero no lo implementes hoy salvo que vayas muy rápido.

Paso 2: parser mínimo
El parser debe comprobar:
Que cada argumento sea un número.
Que quepa en int.
Que no haya duplicados.
Que no haya strings vacíos.
Que en error imprima:
Error
por stderr.
Ejemplos que deberían fallar:
./push_swap 1 2 a
./push_swap 1 2 2
./push_swap 999999999999999999999
./push_swap ""
Ejemplos que deberían ir bien:
./push_swap 3 2 1
./push_swap -1 5 0
./push_swap +3 -2 1

Paso 3: operaciones reales
Antes de ordenar, implementa operaciones. Para MVP mínimo:
void	sa(t_stack **a, int print);
void	ra(t_stack **a, int print);
void	rra(t_stack **a, int print);
Con print = 1, escriben la operación:
sa\n
ra\n
rra\n
Con print = 0, no imprimen nada. Esto te vendrá bien más adelante para ss, rr, rrr.
Después añades:
void	pb(t_stack **a, t_stack **b, int print);
void	pa(t_stack **a, t_stack **b, int print);
Pero para ordenar 2 y 3 números ni siquiera necesitas pa/pb todavía.
Primer algoritmo: ordenar 2 y 3
Para 2 números
si a[0] > a[1] → sa

Para 3 números
Casos:
3 2 1 → sa + rra
3 1 2 → ra
2 3 1 → rra
2 1 3 → sa
1 3 2 → sa + ra
1 2 3 → nada
No lo memorices como magia. Piensa siempre:
sa cambia los dos primeros.
ra manda el primero abajo.
rra sube el último arriba.
Este sort_three es perfecto para empezar porque te obliga a entender las operaciones sin meterte todavía en algoritmos grandes.
